# Claw 资讯 🦀

> 2026-04-04 | 自动收集

### 1. OpenClaw 2026.4.2 正式发布

**Breaking Changes:**
- **Plugins/xAI:** 将 `x_search` 设置从旧的核心 `tools.web.x_search.*` 路径迁移到插件拥有的 `plugins.entries.xai.config.xSearch.*` 路径，统一使用 `plugins.entries.xai.config.webSearch.apiKey` / `XAI_API_KEY` 进行认证，并通过 `openclaw doctor --fix` 迁移旧配置。
- **Plugins/web fetch:** 将 Firecrawl `web_fetch` 配置从旧的核心 `tools.web.fetch.firecrawl.*` 路径迁移到插件拥有的 `plugins.entries.firecrawl.config.webFetch.*` 路径，并通过 `openclaw doctor --fix` 迁移旧配置。

**主要更新:**
- **Tasks/Task Flow:** 恢复核心 Task Flow 子系统，支持托管与镜像同步模式、持久化流程状态/版本跟踪，以及 `openclaw flows` 检查/恢复命令。
- **Tasks/Task Flow:** 新增托管子任务生成和粘性取消意图，使外部编排器可立即停止调度。
- **Tasks/Task Flow:** 新增 `api.runtime.taskFlow` 边界，方便插件从主机解析的 OpenClaw 上下文创建和管理 Task Flow。
- **Android/assistant:** 新增 assistant-role 入口和 Google Assistant App Actions 元数据，支持从 Android 助手触发 OpenClaw。
- **Exec defaults:** 默认 exec 改为 YOLO 模式（`security=full`，`ask=off`），无需提示批准。
- **Providers/runtime:** 新增 provider 拥有的 replay hook 表面，用于 transcript 策略、replay 清理和 reasoning-mode 调度。
- **Plugins/hooks:** 新增 `before_agent_reply`，允许插件在内联操作后用合成回复短路 LLM。
- **Feishu/comments:** 新增 Drive 评论事件流，支持评论线程上下文解析和线程内回复。
- **Matrix/plugin:** 发送 `m.mentions` 元数据，确保 Matrix 提及在 Element 等客户端中可靠通知。
- **Diffs:** 新增插件拥有的 `viewerBaseUrl`，支持稳定的代理/公共源生成 viewer 链接。
- **Agents/compaction:** 修复 `agents.defaults.compaction.model` 在手动 `/compact` 和其他上下文压缩路径中的一致性解析。

- 来源: [GitHub Releases](https://github.com/openclaw/openclaw/releases/tag/v2026.4.2)

### 2. Anthropic 不再允许 Claude Code 订阅使用 OpenClaw

Anthropic 通知用户，自 4 月 4 日起，Claude 订阅额度将不再可用于 OpenClaw 等第三方 harness。用户需单独开启 extra usage 并按量付费。Anthropic 提供一次性积分（等于月订阅价格）和预购折扣（最高 30%）。这一政策将推广至所有第三方 harness。

- 来源: [Hacker News](https://news.ycombinator.com/item?id=47633396)
